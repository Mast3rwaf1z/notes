# Topic: Time, Synchronization and order
**Explain the need for clock synchronization by giving an example scenario. Provide an example synchronization approach for 2 nodes and discuss its advantages and disadvantages**

There is a need for synchronization in the example scenario where you'd be measuring precisely at a given timestamp, you could live with the timestamp being a bit off the desired timestamp but you would want your distributed network of sensors to measure at the exact same time. here it is more important that the time in the nodes correlate to the server node rather than real time. For 2 nodes you'd have a client and a server, the client would ask the server for a timestamp and apply the following formula to set its own time:

$$t_c=t_s-\frac{RTT}{2}$$

where RTT is the round trip time.

**Explain the motivation and realization of a logical clock using an example.**

A logical clock is the implementation of the clock, it can be a vector clock or simple timestamps, the vector clock is characterized by each node having a vector containing the event number of each node that has interacted with this node. in the exercises prior to this one, timestamps was used, which is a logical clock where causality is determined by timestamps sent in communication, while partial ordering is determined using both timestamps and event ids.

# Topic: Distributed exclusion, consistency

**Explain the `Distributed mutual exclusion` problem and compare it to a centralized approach**<br>
The term mutual exclusion describes how no two processes can exist in a critical section of the system at the same time. This means that you can have a database action that can only be done by one process at a time, you would ideally want every process to check the lock before entering the critical section.

first off a centralized approach would mean that entering the critical section is done by first asking a centralized server about whether it can acquire the lock, if it can it can enter, if not it may wait for the lock to be available. the disadvantage of a centralized approach is that the server at the center is a single point of failure for the system, if that server fails, the entire system fails.

The distributed mutual exclusion is on the contrary a way where each node might have a lock and before the node wanting to enter the critical section can enter it must acquire all other nodes locks, otherwise it can assume another process is in the critical section. The disadvantage of this approach is that it is very prone to deadlocks if the system is in partial order, so for distributed mutual exclusion to work it would have to be in a total order. Another disadvantage is that every node will essentially be flooding the network every time they want to enter the critical section, this will be 10 packets for n=10, while for a centralized approach you would only have to ask the server so 1 packet for any n.

**Give an example of a distributed read and write operation sequence and explain two different consistency criteria**<br>
consistency is important for concurrent transactions in a system, it is important to know what the system will respond with.<br>
consistency is always from the perspective of the database, this means that transactions are sequences of operations on a database. let's observe a transaction: $T_1=[e_{11}R(x), e_{12}W(x), e_{13}R(x)]$ and transaction $T_2=[e_{21}W(x), e_{22}W(x), e_{23}R(x)]$. these two transactions can be serializably consistent if the sequence is ordered like this: $[e_{11}, e_{12}, e_{13}, e_{21}, e_{22}, e_{23}]$ and linearly consistent if its ordered by time, lets say the times are something like: <br>
$\begin{array}{|cc|}\hline

The advantage is abstraction. A layer usually functions as a medium for the layers above to communicate with the other end of the communication; for what concerns one layer is that its virtually speaking to the corresponding layer of the target machine, this means that if the layers below a given layer functions properly, the layer above can also function. it is overall easier for a protocol like TCP to be implemented when the developer wouldn't have to be considering lower layers. Taking the TCP example its a layer 4: transport layer protocol managing whether the data is delivered to its destination safely and counteracting packet loss and congestion, it shouldn't have to consider routing, or channels as those are abstracted through the OSI model. The physical layer describes the actual channel the data is being transmitted through, it is not really within scope of this course though, as it often involves a lot of signal processing with modulation or radio waves.

Session, presentation and application layer isn't that important for this course either, the most notable is the application layer as this is where protocols like HTTP exists, this is a protocol to send messages between hosts in a standardized manner using requests. # Topic: ARQ and coding
**Explain the three types of ARQ protocols**<br>
ARQ: Automatic Repeat Request
*   Stop-and-wait

The packet is transmitted and the sender will wait for one of three possibilities: the receiver responds with ACK, the receiver responds with NACK or the receiver does not respond and ARQ reaches a timeout. if NACK or timeout is reached, then the packet is retransmitted. The disadvantage of this protocol is that it does not consider higher data rates or longer RTT's so continuous transmission strategies were also needed:
*   Go-back-N

This uses a sliding window approach, it can send a predefined number of packets before waiting on an ACK, you choose an N before transmitting and it will send N packets, if packet 0 sends an ACK, it will send packet N+1, as the window will now be over 1 - N+1. If a packet loss happens during the transmission the window will return to the lost packet, so if the window was over the previously mentioned packets, and packet 1 is lost, the packets 1 - N+1 will be retransmitted immediately. This method creates more traffic than Stop-and-wait but it is faster than Stop-and-wait.
*   Selective repeat

This is a method where the messages can be sent out of order, and thus the receiver has to order the messages itself. This protocol handles retransmissions by the receiver being able to reject a single packet and request a retransmission without the sender having to retransmit all packets preceding it. the receiver keeps track of the earliest unreceived packet and transmits this number with every ack, the sender will retransmit this packet and continue where it had already reached. The receiver will keep a buffer to reorder the packets and in the case of a buffer overflow the receiver will also request for retransmissions.

**Explain the difference between error correction and error detection. Give examples when it is preferable to use one or another, or a combination of them**<br>
error correction often happens in the actual packets, if there is an error in the transmission - an example could be a faulty channel, then the packet may have a mechanism to reconstruct some data that has been lost. This check might be a parity check where the packet is checked with a parity bit or a checksum, if this parity check fails the packet may need to be retransmitted. an example of error correction could be FEC: Forward Error Correction, this is where the transmitter sends a lot of redundant information to the receiver. Then the receiver only recognizes the portion of this data that contains no apparent errors. The disadvantage is that the packets gets much larger in this way because for a high reliability you must have a very long error correction code.

Error detection, and more notably cyclic redundancy check is defined by defining the messages transmitted between the transmitter and receiver as polynomials, that means for a byte like 10110001 would have a polynomial defined like this:
$$M(x)=(1\cdot x^7)+(0\cdot x^6)+(1\cdot x^5)+(1\cdot x^4)+(0\cdot x^3)+(0\cdot x^2)+(0\cdot x^1)+ (1\cdot x^0)$$
$$M(x)=x^7+x^5+x^4+1$$
and doing some complicated math to calculate whether the data in the packet is correct, this is useful as the only extra data to be added to the packets are usually a check value, a value used in the calculations. using this check a remainder will be returned which symbolizes the error rate of the packet, a low remainder means the packet is relatively error free. there might also be attached a residue constant which symbolizes the expected remainder. if the values does not match, the data contains an error.

# Topic: MAC Protocols
**Give examples of MAC protocols with static channel allocation. Discuss their advantages and disadvantages**<br>
if we observe a protocol like FDMA (Frequency division multiple access), its a protocol where we divide a frequency channel into smaller segments; dividing it into smaller bandwidths and letting the users send as frequently as they want on that bandwidth. This is not very efficient in most configurations as most traffic is bursty, letting the channel be idle most of the time while when one user wants to send a lot of data at once they will be limited by the limited bandwidth. this protocol can be implemented with static channel allocation such that the bandwidth is divided once and never afterwards.

another example would be TDMA (Time Division Multiple Access) where a time period would be split evenly between all users, the advantage here is that every user will be able to use the full bandwidth during their time but the channel might be idle most of the time with this method, this can also be implemented with static channel allocation such that all users gets their time slot once. This is a big disadvantage as if one user leaves, that time slot will always be idle afterwards.

**Explain the principle behind a Random Access class of MAC protocols. Give an example of a such a protocol**<br>
A random access protocol could be the Aloha protocol

The aloha protocol works by transmitting immediately on the channel, if the channel is busy the protocol will wait for a random period and try to transmit on the channel again. This would be a very messy protocol to use as there will be a large amount of packet loss but its also one where the communication overhead between nodes will be very low.

Aloha isn't very efficient as it would peak at a throughput of about 40% of the potential throughput, CSMA is an alternate version that is much more efficient

**Explain main features of Carrier Sense Multiple Access (CSMA) protocol, including the difference between non-persistent and 1-persistent versions**<br>
so the main difference between CSMA and aloha would be that CSMA does listen to the channel before transmitting, that means it will check whether the channel is busy before transmitting itself, it is not that different to aloha, it will treat a concurrent transmission as a collision, but the difference is that the colliding client will never transmit, letting the first client finish their transmission without a collision, effectively turning aloha's low 40% throughput into a potential 90% throughput. This is because in aloha both transmissions would be lost in case of a collision while in CSMA only the 2nd transmission would be lost.

Non-persistent waits for a random period before attempting to transmit again

1-persistent waits until the channel is no longer busy to transmit, this might be prone to collisions though, as during one transmission there might be multiple clients waiting to transmit. at this collision the clients may resort to wait by a random amount like non-persistent CSMA.
